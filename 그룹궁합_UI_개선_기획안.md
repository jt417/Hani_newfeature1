# 그룹/커플 궁합 방 만들기 UI 개선 기획안

## 1. 문서 목적

현재 HANI의 그룹 궁합 방 만들기 화면은 기능 흐름은 존재하지만, 실제 사용 경험은 `너무 비어 보이고`, `기대감이 약하며`, `모임 종류가 부족하고`, `결과 화면이 단조롭다`.

이 문서는 현재 [`src/features/GroupRoom.jsx`](/Users/jaylee/Downloads/HANI%20%E1%84%8E%E1%85%AE%E1%84%80%E1%85%A1%E1%84%80%E1%85%B5%E1%84%82%E1%85%B3%E1%86%BC/src/features/GroupRoom.jsx) 기준으로 그룹/커플 궁합 방 만들기 경험 전체를 개선하기 위한 기획안이다.

목표는 아래 4가지다.

- 첫 진입에서 `해보고 싶다`는 감정을 만들기
- 방 만들기 과정을 더 풍부하고 직관적으로 만들기
- 그룹 종류를 실제 사용자 맥락에 맞게 넓히기
- 결과 화면을 단순 점수 페이지가 아니라 `공유하고 싶은 결과`로 만들기

---

## 2. 현재 화면 진단

현재 구조는 아래 단계로 구성되어 있다.

- landing
- qr_scan
- create
- join_input
- lobby
- analyzing
- result_public
- result_private
- result_advice
- save_complete

### 2.1 핵심 문제

#### A. 첫 화면이 너무 비어 있다

- 현재 랜딩은 아이콘, 제목, 짧은 설명, 버튼만 있다.
- "왜 이 기능을 써야 하는지"와 "어떤 재미를 주는지"가 약하다.
- 친구/커플과 함께 할 때 느껴지는 설렘이나 기대감이 시각적으로 거의 없다.

#### B. 방 만들기 화면의 정보량이 너무 적다

- 현재 `create` 단계는 사실상 모임 종류 3개만 선택하는 화면이다.
- 방 이름도 없고, 방 성격도 없고, 인원 규모 맥락도 없어서 너무 썰렁하다.
- 사용자가 "내가 뭘 만들고 있는지" 감각이 약하다.

#### C. 모임 종류가 현실 사용 맥락에 비해 부족하다

현재 선택지는:

- 동네 친구
- 대학 친구
- 운동 모임

이 정도로는 실제 20~30대 여성 사용 패턴을 담지 못한다.

#### D. 로비 화면이 대기실 느낌이 약하다

- 지금은 참여자 리스트만 있고, 방에 대한 개성이 거의 없다.
- 누가 아직 안 들어왔는지, 어떤 분위기로 분석되는지, 예상 질문이 뭔지 같은 기대감 장치가 없다.

#### E. 결과 화면이 단일 점수 중심이라 감정적 보상이 약하다

- `82점`, 역할 배지 몇 개, 개인 결과 버튼 정도만 있다.
- 공유 욕구를 만들 정도의 스토리텔링이 부족하다.
- 공용 결과와 개인 결과 사이 연결도 매끄럽지 않다.

#### F. 커플 궁합과 그룹 궁합이 분리되어 보이지 않는다

- 현재 구조상 사실상 `그룹 궁합` 하나로만 읽힌다.
- `커플/썸/연애 궁합`은 전혀 다른 기대를 가지기 때문에 별도 진입 설계가 필요하다.

## 3. 개선 방향 한 줄 정의

이 기능은 단순히 "방을 만드는 도구"가 아니라, `우리 관계를 이벤트처럼 해석해보는 경험`이 되어야 한다.

즉, 다음처럼 바뀌어야 한다.

- 기능 중심 UI -> 분위기 있는 참여형 경험
- 빈 폼 -> 테마가 있는 방 생성
- 단순 점수 결과 -> 역할/케미/주의 포인트가 담긴 리포트

---

## 4. 타깃 관점에서 기대하는 경험

20~30대 여성 사용자가 이 기능에서 기대하는 것은 단순한 숫자 점수가 아니다.

### 4.1 기대 포인트

- 우리 사이를 재미있게 정리해보고 싶다
- 친구/커플과 함께 볼 때 반응이 나와야 한다
- 너무 무섭거나 단정적이지 않았으면 좋겠다
- 예쁘고 공유 가능한 결과였으면 좋겠다
- 그룹이면 "누가 분위기 메이커인지", 커플이면 "우리가 어떤 케미인지" 알고 싶다

### 4.2 UX가 줘야 하는 감정

- 시작 전: 궁금함
- 참여 중: 기대감
- 결과 확인: 공감 + 웃김 + 납득
- 저장 후: 내 관계맵에 넣고 싶음

## 5. 전체 UX 재설계 제안

### 5.1 구조 개편

현재는 `landing -> create -> lobby -> result`로 매우 평면적이다.

추천 구조는 아래다.

1. 진입 선택
2. 방 컨셉 설정
3. 참여자 모으기
4. 분석 미리보기
5. 공용 결과
6. 개인 결과
7. 저장/공유

### 5.2 그룹과 커플 분리 진입

첫 화면에서 아래 2갈래를 분리해야 한다.

- 그룹 궁합
- 커플/썸 궁합

이 둘은 결과 톤도 달라야 한다.

#### 그룹 궁합

- 우리 모임 분위기
- 각자 역할
- 대화 흐름
- 갈등 포인트

#### 커플/썸 궁합

- 감정 온도
- 관계 리듬
- 서로의 표현 차이
- 잘 맞는 포인트 / 어긋나는 포인트

## 6. 랜딩 화면 개선안

### 6.1 현재 문제

- 너무 비어 있다
- 기능 설명이 평면적이다
- 진입 이유가 약하다

### 6.2 개선 목표

- 첫 화면만 보고도 "한 번 해볼까?"가 나와야 한다
- 그룹/커플 각각의 기대 결과가 보여야 한다

### 6.3 추천 구성

#### 상단 히어로 카드

- 큰 타이틀
- 부제
- 샘플 결과 예시 2~3개

예시 카피

- `우리 모임의 진짜 분위기, 같이 보면 더 재밌어요`
- `누가 연결자고, 누가 텐션을 끌어올리는지 한눈에`

#### 모드 선택 카드

- 그룹 궁합 카드
- 커플/썸 궁합 카드

각 카드에는 아래 포함

- 대표 아이콘/일러스트
- 한 줄 설명
- 결과 예시 태그

예시

- 그룹 궁합: `연결자`, `분위기메이커`, `갈등 주의`
- 커플 궁합: `설렘형`, `속도 차`, `표현 방식 차이`

#### 하단 액션

- 방 만들기
- QR로 입장
- 코드로 입장

### 6.4 비주얼 방향

- 지금보다 훨씬 풍성한 카드형 랜딩
- 결과 예시 미리보기를 배경에 살짝 노출
- 단순 아이콘 1개보다, 관계를 암시하는 장면형 일러스트가 더 적합

## 7. 방 만들기 화면 개선안

### 7.1 현재 문제

- 모임 종류만 고르는 수준
- 너무 비어 있고 실사용 맥락이 부족함

### 7.2 개선 목표

- 방을 `생성`하는 느낌이 나야 한다
- 모임 성격을 미리 정하면 결과도 더 기대되게 해야 한다

### 7.3 입력 항목 제안

#### 필수

- 분석 모드
  - 그룹
  - 커플/썸
- 모임 종류
- 방 이름

#### 선택

- 인원 규모
- 현재 분위기
- 궁금한 포인트
- 공개 범위

### 7.4 모임 종류 확장안

현재보다 훨씬 넓혀야 한다.

#### 친구/지인

- 동네 친구
- 대학 친구
- 고등학교 친구
- 베프 모임
- 여행 멤버
- 술친구
- 생일 모임

#### 직장/커리어

- 회사 팀
- 프로젝트 TF
- 동기 모임
- 스터디 팀
- 사이드 프로젝트
- 창업 멤버

#### 취미/라이프스타일

- 운동 모임
- 러닝크루
- 독서모임
- 취미 클래스
- 덕질 모임
- 봉사 모임

#### 가족/생활

- 가족
- 사촌 모임
- 자매/형제
- 자취 메이트

#### 연애/커플

- 썸
- 연인
- 장기 연애
- 재회 고민
- 결혼 준비

### 7.5 궁금한 포인트 선택

결과 기대감을 높이려면 질문 기반 선택이 좋다.

예시

- 누가 분위기 메이커인지 궁금해요
- 우리 모임이 왜 어색해지는지 알고 싶어요
- 누가 갈등을 완충하는지 알고 싶어요
- 우리 커플의 표현 방식 차이가 궁금해요
- 썸에서 누가 더 천천히 다가가는 타입인지 알고 싶어요

### 7.6 추천 UI

- 상단 진행 바
- 방 정보 카드
- 선택 칩 + 큰 카드 혼합
- 하단 고정 CTA

## 8. 로비(대기실) 화면 개선안

### 8.1 현재 문제

- 참여자 리스트만 있음
- 대기실 감정이 없음
- 방의 컨셉과 기대 결과가 비어 있음

### 8.2 개선 목표

- "방에 들어와 있는 느낌"을 만들어야 함
- 결과 기대감을 계속 키워야 함

### 8.3 추천 구성

#### 상단 방 카드

- 방 이름
- 모임 종류
- 인원 수
- 분석 포인트 요약

예시

- `주말 러닝크루`
- `운동 모임 · 4명`
- `오늘 궁금한 포인트: 누가 텐션을 끌어올리는지`

#### 참여자 영역

- 프로필 카드형 리스트
- 입장 상태
  - 입장 완료
  - 대기 중
  - 정보 입력 중

#### 미리보기 영역

- 분석에서 나올 항목 예고

예시

- `분위기 메이커`
- `갈등 완충자`
- `대화 리듬`
- `속마음은 각자만 확인 가능`

#### 초대 영역

- 초대 링크
- QR
- 코드 복사

### 8.4 추가하면 좋은 장치

- 랜덤 티저 문구
  - `이 모임엔 의외의 연결자가 숨어 있을지도 몰라요`
  - `겉으로 조용한 사람이 분위기를 잡는 경우도 있어요`

## 9. 분석 중 화면 개선안

### 9.1 현재 문제

- 너무 짧고 단순함
- 기다리는 동안 재미가 없음

### 9.2 개선 목표

- 기다리는 2~3초도 경험으로 느껴지게 하기

### 9.3 추천 구성

- 분석 단계 표시
  - 관계 흐름 스캔 중
  - 역할 분포 계산 중
  - 대화 케미 정리 중
  - 개인 결과 분리 중

- 부드러운 모션
- 참여자 아바타가 떠다니는 연출
- 하단 한 줄 카피

예시

## 10. 공용 결과 화면 개선안

### 10.1 현재 문제

- 점수와 역할 배지 몇 개만 있음
- 한 번 보고 끝나는 화면 느낌

### 10.2 개선 목표

- 공용 결과만 봐도 "오, 맞아"가 나와야 함
- 그룹이면 웃기고 공감돼야 하고, 커플이면 섬세해야 한다

### 10.3 추천 결과 구조

#### 상단 요약 카드

- 총평 한 줄
- 그룹/커플 분위기 명칭
- 결과 배경 일러스트 또는 무드 그래픽

예시

- `말이 끊기지 않는 활기형 모임`
- `속도 차는 있지만 정이 깊은 커플`

#### 핵심 지표 3개

- 분위기 온도
- 안정감
- 갈등 민감도

#### 역할/케미 카드

##### 그룹

- 분위기 메이커
- 연결자
- 완충자
- 현실 담당

##### 커플

- 표현 주도형
- 감정 축적형
- 속도 조절형
- 안정감 제공형

#### 주의 포인트

- 공용으로 볼 수 있는 선에서만
- 너무 세게 말하지 않기

예시

- `대화가 빨라질수록 한 사람이 소외감을 느낄 수 있어요`
- `장난이 길어지면 감정 온도 차가 커질 수 있어요`

#### 하단 CTA

- 내 개인 결과 보기
- 결과 카드 저장
- 친구에게 공유

## 11. 개인 결과 화면 개선안

### 11.1 현재 문제

- 정보량이 너무 적다
- 개인화 깊이가 부족하다

### 11.2 개선 목표

- "이 모임에서의 나"를 선명하게 느끼게 해야 함

### 11.3 추천 구조

- 내 역할 한 줄
- 내가 편한 사람 / 조심할 사람
- 내가 분위기에 주는 영향
- 행동 가이드

예시

- `당신은 이 모임의 속도를 끌어올리는 사람`
- `대화를 먼저 여는 편이라 분위기 전환에 강해요`
- `다만 피곤한 날엔 직설적으로 들릴 수 있어요`

### 11.4 행동 가이드 예시

- 오늘은 먼저 말을 꺼내면 분위기가 풀릴 수 있어요
- 상대가 반응이 느려도 거리감으로 단정하지 않는 편이 좋아요
- 장난보다 리액션이 더 중요한 날이에요

## 13. 그룹/커플별 톤 차이 제안

### 13.1 그룹 궁합 톤

- 경쾌함
- 역할 중심
- 분위기 중심
- 밈처럼 공유 가능한 문구

예시

- `이 모임은 조용한 듯 은근 텐션이 센 편`

### 13.2 커플/썸 궁합 톤

- 더 섬세하고 부드럽게
- 관계 온도와 리듬 중심
- 단정적이지 않게

예시

- `표현 방식은 다르지만, 정은 오래 가는 조합`

## 14. 추천 모임 종류 최종안

출시 초반에는 너무 많아도 복잡하므로 아래 정도가 적당하다.

### 14.1 그룹

- 동네 친구
- 대학 친구

## 18. 최종 제안

지금의 그룹 궁합 방 만들기 UI는 기능은 있지만, `경험`은 약하다.

가장 중요한 개선 포인트는 아래다.

1. 첫 화면에서 기대감을 높이기
2. 그룹과 커플을 명확히 분리하기
3. 방 만들기 과정에 컨셉과 질문을 넣기
4. 대기실에 기대감을 심기
5. 결과를 점수 페이지가 아니라 관계 리포트처럼 만들기

결론적으로 이 기능은 `입장 -> 생성 -> 결과 확인`이 아니라, `우리 관계를 이벤트처럼 같이 해석해보는 경험`으로 바뀌어야 한다.

## 21. 결론

현재 그룹 궁합 방 만들기 UI의 가장 큰 문제는 화면이 비어 있는 것이 아니라, `기대감과 이야기 구조가 비어 있는 것`이다.

따라서 개선의 핵심은 단순히 요소를 더 추가하는 것이 아니라 아래를 만드는 것이다.

- 방에 들어가고 싶은 이유
- 같이 기다릴 만한 기대감
- 결과를 보고 반응할 만한 포인트
- 저장하고 공유하고 싶은 마무리

이 기준으로 보면, 가장 먼저 손봐야 할 곳은 `랜딩`, `방 만들기`, `로비`, `공용 결과` 4개다.
